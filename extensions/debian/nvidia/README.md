# Расширение NVIDIA

Устанавливает драйверы NVIDIA для виртуальных машин Bisquite с поддержкой GPU passthrough.

## Поддерживаемые системы

- **Debian**: 12 (Bookworm), 13 (Trixie)
- **Ubuntu**: 20.04 (Focal), 22.04 (Jammy), 24.04 (Noble)

## Возможности

- Автоматическое определение ОС и версии
- Добавление необходимых репозиториев (contrib, non-free, non-free-firmware для Debian; graphics-drivers PPA для Ubuntu)
- Автоматическая установка рекомендованных драйверов через `nvidia-detect` (Debian, опционально) или `ubuntu-drivers` (Ubuntu)
- Автоматическая блокировка модуля `nouveau` для предотвращения конфликтов
- Обновление initramfs для применения блокировки nouveau
- Retry-механизм для установки пакетов
- Поддержка установки в chroot окружении (использует метапакет `linux-headers-amd64` вместо `uname -r`)

## Требования

- Debian 12/13 или Ubuntu 20.04/22.04/24.04
- Доступ к интернету при сборке
- Минимум 500 МБ свободного места
- Для Debian: необходим доступ к репозиториям contrib и non-free-firmware
- Для Ubuntu: необходим доступ к PPA репозиториям

## Состав

- `install.sh` — основной скрипт установки драйверов NVIDIA

## Интеграция в VMFILE

```bash
COPY_IN ../files/nvidia:/opt/vmsetup/
RUN_COMMAND chmod +x /opt/vmsetup/nvidia/*.sh
RUN_COMMAND /opt/vmsetup/nvidia/install.sh
```

## Как это работает

1. **Определение ОС** — скрипт использует `lsb_release` для определения дистрибутива и версии
2. **Настройка репозиториев**:
   - **Debian**: добавляет `contrib`, `non-free` и `non-free-firmware` в существующие репозитории (с fallback на добавление новых строк)
   - **Ubuntu**: добавляет PPA репозиторий graphics-drivers для соответствующей версии с современным способом добавления GPG ключей
3. **Установка драйверов**:
   - **Debian**: пытается установить `nvidia-detect` (опционально, если недоступен — продолжает без него), затем устанавливает `nvidia-driver`, `linux-headers-amd64` (метапакет), `build-essential` и `dkms`
   - **Ubuntu**: использует `ubuntu-drivers autoinstall` для автоматической установки рекомендованного драйвера (с fallback на ручную установку)
4. **Блокировка nouveau**:
   - Создаёт `/etc/modprobe.d/blacklist-nouveau.conf` с блокировкой nouveau
   - Обновляет initramfs для применения блокировки при загрузке
5. **Завершение** — драйверы установлены, nouveau заблокирован, требуется перезагрузка для применения изменений

## Проверка

После перезагрузки проверьте установку:

```bash
# Проверка загруженных модулей NVIDIA
lsmod | grep nvidia

# Проверка статуса драйверов
nvidia-smi

# Проверка версии драйвера
nvidia-driver-version

# Убедитесь, что nouveau НЕ загружен
lsmod | grep nouveau  # должно быть пусто

# Проверка блокировки nouveau
cat /etc/modprobe.d/blacklist-nouveau.conf
```

## Диагностика

Если драйверы не установились или не работают:

```bash
# Проверка установленных пакетов
dpkg -l | grep nvidia

# Проверка логов установки
journalctl -xe

# Для Debian: проверка определения драйвера
nvidia-detect

# Для Ubuntu: проверка доступных драйверов
ubuntu-drivers list

# Проверка загруженных модулей
lsmod | grep -E "nvidia|nouveau"

# Проверка dmesg на ошибки
dmesg | grep -i nvidia
dmesg | grep -i nouveau
```

Распространённые проблемы:

- **Репозитории недоступны** — проверьте DNS/прокси и доступность репозиториев
- **nvidia-detect не найден** — это нормально для некоторых версий Debian, скрипт продолжит установку без него
- **Драйвер не установился** — проверьте логи установки, возможно требуется ручная установка конкретной версии драйвера
- **Ошибки при компиляции модулей** — убедитесь, что установлены `linux-headers-amd64` и `build-essential`. Скрипт автоматически использует метапакет для корректной работы в chroot
- **nouveau загружен вместо nvidia** — проверьте, что `/etc/modprobe.d/blacklist-nouveau.conf` существует и содержит блокировку nouveau. Скрипт автоматически обновляет initramfs, но если проблема остаётся, выполните `update-initramfs -u` и перезагрузите систему
- **nvidia-smi не работает** — убедитесь, что модули nvidia загружены (`lsmod | grep nvidia`), nouveau не загружен (`lsmod | grep nouveau` должно быть пусто), и выполните перезагрузку после установки

## Примечания

- Скрипт работает без `sudo`, так как запускается от root в chroot окружении
- Для работы NVIDIA драйверов требуется GPU passthrough в гипервизоре
- После установки драйверов требуется перезагрузка системы для загрузки модулей nvidia и применения блокировки nouveau
- Скрипт автоматически блокирует nouveau и обновляет initramfs — это критически важно для корректной работы проприетарных драйверов
- Используется метапакет `linux-headers-amd64` вместо `linux-headers-$(uname -r)` для корректной работы в chroot окружении, где `uname -r` может вернуть ядро хоста
- В некоторых случаях может потребоваться ручная установка конкретной версии драйвера
