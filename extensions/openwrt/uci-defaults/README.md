# Скрипты авторесайза корневого диска

Каталог содержит uci-defaults-скрипты, которые выполняются OpenWrt один раз при первом полном старте и служат для автоматического увеличения системного раздела после деплоя облачного образа.

## Содержимое

### `70-rootpt-resize`
- Определяет блочное устройство, на котором находится `root`, и расширяет соответствующий раздел до 100 % доступного места с помощью `parted`.
- Создаёт флаг `/etc/rootpt-resize`, чтобы повторно не выполнять операцию, и инициирует перезагрузку.
- Требует наличия `parted` и механизма блокировок `lock` (в OpenWrt уже есть в busybox).

### `80-rootfs-resize`
- Запускается после успешного изменения таблицы разделов (`/etc/rootpt-resize` уже существует) и расширяет файловую систему `ext4` с помощью `resize2fs`.
- При необходимости подключает root-раздел через временный `loop`-device, чтобы выполнить `resize2fs` даже когда `overlay` уже смонтирован.
- Создаёт флаг `/etc/rootfs-resize` и выполняет финальную перезагрузку.

## Как подключить при сборке образа

Во время сборки VM-образа добавьте оба файла в `/etc/uci-defaults` и сделайте их исполняемыми. В `openwrt.vmfile` это выглядит так:

```dockerfile
UPLOAD ./files/uci-defaults/70-rootpt-resize:/etc/uci-defaults/70-rootpt-resize
UPLOAD ./files/uci-defaults/80-rootfs-resize:/etc/uci-defaults/80-rootfs-resize
RUN_COMMAND chmod +x /etc/uci-defaults/70-rootpt-resize /etc/uci-defaults/80-rootfs-resize
RUN_COMMAND "echo '/etc/uci-defaults/70-rootpt-resize' >> /etc/sysupgrade.conf"
RUN_COMMAND "echo '/etc/uci-defaults/80-rootfs-resize' >> /etc/sysupgrade.conf"
```

Добавление путей в `/etc/sysupgrade.conf` гарантирует, что скрипты сохранятся при последующих `sysupgrade` и останутся доступными для будущих развёртываний базового образа.

## Как использовать внутри VM

1. После первого запуска OpenWrt автоматически выполнит `uci-defaults`-скрипты. Ожидаются две последовательные перезагрузки: первая после расширения раздела, вторая после расширения файловой системы.
2. Процесс можно мониторить в консоли через `logread -f | grep root-resize`.
3. Если автоматика не нужна, создайте пустые файлы `/etc/rootpt-resize` и `/etc/rootfs-resize` ещё на стадии сборки — тогда скрипты завершатся сразу.
4. Проверить результат можно командой `lsblk` или `df -h` после второго перезапуска.

Таким образом, готовый образ автоматически занимает всё предоставленное дисковое пространство без ручного вмешательства в гипервизоре.
